# 텍스트 추론 플레이북

## 사용 조건

- 입력이 주로 텍스트
- 답이 조건, 논리, 순서, 또는 꼼꼼한 읽기에 달려 있음

## 기본 경로

`Orchestrator → Solver → Verifier`

선택적:
- 후보가 두 개 이상 살아남으면 `Critic` 추가
- 문제 지문이 길거나 지저분할 때만 `Extractor` 추가

## 주요 목표

답을 쫓기 전에 문제를 컴팩트한 조건 집합으로 변환.

## Human Checkpoint

최종 모호성이 규칙 기반이 아닌 판단 기반일 때만 사용.

## 선택적 코딩

허용 범위:
- 조합론 검증
- 테이블 정리
- 계산 또는 열거

## Compressed Mode

- 문제를 한 문장으로 재기술
- 가장 강한 제약조건 3개 나열
- 지배적 후보 1개 생성
- 짧은 Verifier 패스 실행

## 흔한 실패 패턴

- 지문 속에 숨겨진 작은 조건 하나를 놓침
- 유창하지만 얕은 모델 답변을 신뢰함

# 이미지 / OCR 플레이북

## 사용 조건

- 문제가 스크린샷, 스캔 텍스트, 차트, 다이어그램, 노이즈 이미지에 의존

## 기본 경로

`Orchestrator → Extractor → Solver → Verifier`

선택적:
- 모호한 영역에서 제출 전 `Human Checkpoint` 추가
- OCR과 해석이 불일치하면 `Critic` 추가

## 주요 목표

추출 품질과 추론 품질을 분리.

## Human Checkpoint

강력 권장 조건:
- OCR에 깨진 문자가 있을 때
- 두 가지 시각적 해석이 모두 그럴듯할 때
- 정확한 최종 텍스트가 중요할 때

## 선택적 코딩

허용 범위:
- OCR 후처리
- 중복 제거
- 토큰 정리
- 좌표 또는 테이블 정규화

## Compressed Mode

- 최종 답에 관련된 영역만 추출
- 이미지 전체를 과도하게 파싱하지 않음
- 정확한 최종 문자열 또는 숫자 검증

## 흔한 실패 패턴

- 오염된 OCR 결과로 풀이 진행
- 추측한 텍스트를 추출된 텍스트에 섞음

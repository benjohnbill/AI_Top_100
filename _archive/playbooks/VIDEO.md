# 영상 플레이북

## 사용 조건

- 답이 움직임, 순서, 타이밍, 장면 전환, 또는 프레임 단위 이벤트에 달려 있음

## 기본 경로

`Orchestrator → Extractor → Solver → Verifier`

선택적:
- 장면 해석이 불안정하면 `Critic` 추가
- 시각적 모호성이나 이벤트 순서 분쟁이 있으면 `Human Checkpoint` 추가

## 주요 목표

추론 전에 움직이는 소스를 안정적인 타임라인으로 변환.

## Human Checkpoint

사용 조건:
- 이벤트 경계가 불명확할 때
- 프레임마다 객체 정체성이 변할 때
- 작은 시각적 단서가 답을 결정할 때

## 선택적 코딩

허용 범위:
- 타임스탬프 기록
- 프레임 목록 정리
- 이벤트 테이블 구축

## Compressed Mode

- 핵심 장면에만 집중
- 촘촘한 트랜스크립트 대신 희소 타임라인 사용
- 최종 순서 또는 개수 검증

## 흔한 실패 패턴

- 시간 앵커 없이 모호한 요약으로 추론
- 결정적인 한 프레임을 놓침

# 모델링 / 분석 플레이북

## 사용 조건

- 피처 설계, 베이스라인 구축, 순위 결정, 또는 반복 제출이 중요한 과제

## 기본 경로

`Orchestrator → Solver → Verifier → Submission Loop`

선택적:
- 과적합이나 맹점이 우려되면 `Critic` 추가

## 주요 목표

빠르게 안정적인 베이스라인 도달, 이후 통제된 변경으로 개선.

## Human Checkpoint

대규모 전략 전환 전 또는 피드백 패턴이 불명확할 때 사용.

## 선택적 코딩

일반적으로 유용한 범위:
- 파싱
- 피처 생성
- 배치 평가
- 제출 포맷팅
- 결과 비교

## Compressed Mode

- 유효한 베이스라인을 먼저 제출
- 한 번에 변수 하나씩 개선
- 점수를 높인 변경은 유지

## 흔한 실패 패턴

- 신뢰할 수 있는 베이스라인 없이 복잡한 아이디어 추구
- 제출 사이에 너무 많은 변수를 동시에 변경

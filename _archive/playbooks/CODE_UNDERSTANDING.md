# 코드 이해 플레이북

## 사용 조건

- 문제가 코드 동작을 읽고, 추적하고, 설명하는 것에 집중

## 기본 경로

`Orchestrator → Solver → Verifier`

선택적:
- 숨겨진 엣지 케이스가 있으면 `Critic` 추가
- 코드가 길어서 먼저 분절이 필요하면 `Extractor` 추가

## 주요 목표

코드를 구조, 흐름, 결정적인 상태 변화로 축약.

## Human Checkpoint

거의 사용하지 않음. 대부분 트레이싱과 검증으로 해결 가능.

## 선택적 코딩

허용 범위:
- 경량 트레이싱
- 소규모 입력 시뮬레이션
- 작은 도메인의 브루트포스

## Compressed Mode

- 코드를 블록 단위로 요약
- 출력을 제어하는 분기 또는 루프 하나 파악
- 최소 트레이스로 검증

## 흔한 실패 패턴

- 실제 출력 동작을 놓치면서 코드를 과도하게 설명
- 분기 하나, 경계값 하나, 변이 하나를 잊음
